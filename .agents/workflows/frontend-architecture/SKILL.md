# Frontend Architecture ‚Äî Feature-Based

## Overview

`resources/js` is organised around **features** (business domains), not file
types. Every directory has one clear responsibility. Do not invent new top-level
directories without discussion.

## Directory Contracts

### `components/ui/`
- **Source:** shadcn/ui primitives (generated via `npx shadcn add ‚Ä¶`).
- **Rules:**
  - Zero business logic.
  - Fully generic and reusable across all features.
  - Files are **kebab-case** (`button.tsx`, `input-otp.tsx`).
  - Never import from `features/`.

### `components/common/`
- Shared building blocks that are **not** shadcn primitives and **not**
  feature-specific.
- Examples: `AppLogo`, `ErrorBoundary`, `Breadcrumbs`, `Heading`,
  `InputError`, `TextLink`, `UserInfo`, motion/animation components.
- Files are **PascalCase** (`AppLogo.tsx`).

### `components/layouts/`
- Sub-components of the app shell used **inside** layout files.
- Examples: `AppHeader`, `AppShell`, `AppContent`, `GlobalSearch`,
  `UserMenuContent`.
- Files are **PascalCase**.

### `features/<name>/components/`
- Components that are **specific to one feature**.
- Each feature directory mirrors its `pages/<name>` counterpart.
- Allowed to import from `components/ui`, `components/common`, `hooks`,
  `lib`, and `types`.
- Must **not** import from another feature's `components/` directory.
  Cross-feature reuse ‚Üí move the component to `components/common/`.

### `features/<name>/hooks/` *(optional, if feature has many hooks)*
- Feature-scoped hooks (e.g., `use-book-filters.ts` inside `features/catalog/hooks/`).

### `features/<name>/types.ts` *(optional)*
- Types used exclusively within the feature (e.g., `features/kiosk/types.ts`).

### `hooks/`
- Global custom hooks with **no** JSX (`*.ts` preferred, `*.tsx` only when
  returning JSX).
- Name format: `use-<noun>.ts` (kebab-case).
- Must be pure (no side-effects at module load time).

### `layouts/`
- Full-page Inertia layout components.
- Referenced in `app.tsx` layout resolver.
- Files are **PascalCase** (`AppLayout.tsx`).

### `lib/`
- Pure TypeScript functions ‚Äî no React, no Inertia.
- `utils.ts` for `cn()`, `toUrl()`, currency, date formatters, etc.
- Add more files as the library grows (`date.ts`, `currency.ts`, ‚Ä¶).

### `pages/`
- **Thin** Inertia entry-points. Should contain minimal JSX ‚Äî delegate
  rendering to `features/<name>/components/`.
- One file = one URL. Folders mirror URL segments (`pages/books/show.tsx`
  ‚Üí `/books/:book`).
- Files are **kebab-case** (Inertia convention: `show.tsx`, `create.tsx`).

### `types/`
- Shared TypeScript interfaces and type aliases.
- Feature-only types live inside `features/<name>/types.ts`.

## Naming Conventions

| Item | Convention | Example |
|---|---|---|
| React component file | PascalCase | `BookCard.tsx` |
| Inertia page file | kebab-case | `show.tsx`, `forgot-password.tsx` |
| shadcn UI primitive | kebab-case | `button.tsx`, `input-otp.tsx` |
| Custom hook file | `use-` + kebab-case | `use-clipboard.ts` |
| Utility file | kebab-case | `utils.ts`, `date.ts` |
| Type file | kebab-case | `auth.ts`, `navigation.ts` |
| React component (name) | PascalCase | `<BookCard />` |
| Hook (name) | camelCase | `useClipboard()` |
| Utility function | camelCase | `formatCurrency()` |
| Constant | SCREAMING_SNAKE_CASE | `MAX_BORROW_DAYS` |

## Import Path Aliases

Always use the `@/` alias. **Never use relative paths** that traverse more
than one level up (`../../`).

```ts
// ‚úÖ Correct
import { BookCard } from '@/features/catalog/components/BookCard';
import { cn } from '@/lib/utils';
import { useClipboard } from '@/hooks/use-clipboard';
import type { Book } from '@/types';

// ‚ùå Wrong
import { BookCard } from '../../components/catalog/BookCard';
```

---

## Page Architecture Pattern

Pages must be thin. Move complex JSX to feature components.

```tsx
// pages/books/show.tsx ‚Äî ‚úÖ thin page
import { BookDetail } from '@/features/catalog/components/BookDetail';
import type { Book } from '@/types';

interface Props {
    book: Book;
}

export default function Show({ book }: Props) {
    return <BookDetail book={book} />;
}
```

## Adding a New Feature Checklist

When creating a new business feature (e.g., `loans`):

1. Create `features/loans/components/` for feature components.
2. Create `features/loans/hooks/` if the feature needs dedicated hooks.
3. Create `features/loans/types.ts` if the feature has private types.
4. Add page(s) under `pages/loans/`.
5. Register global shared types in `types/index.ts`.
6. Register new layout resolution logic in `app.tsx` if needed.
